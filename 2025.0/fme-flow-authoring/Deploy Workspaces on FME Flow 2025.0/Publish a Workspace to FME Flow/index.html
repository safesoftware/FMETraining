Publishing Wizard

In FME Workbench, the publishing process to transfer workspaces to FME Flow is completed using the publishing wizard. The publishing wizard can be opened by clicking Publish in the toolbar

...or from File > Publish to FME Flow on the top menu. 

On the first page, you select or create your FME Flow Connection. The options here are the same as those in the FME Flow Connection toolbar. 

Then you select the repository to house your workspace. Repositories are like folders on FME Flow; they hold and organize workspaces and their related data. The Existing Workspace List will update with the workspaces already in your selected repository. 

Next, you have the option to change the workspace name. If you input the name of a workspace that already exists on FME Flow in that repository, a warning message will ask you if you'd like to overwrite the existing workspace. 

You also have the option to upload data files and select which files to upload. The data is uploaded to the repository where the workspace resides by default. You will learn more about managing source data in **the next module**. 

If your workspace uses a Web or Database Connection, the next step allows you to select which connections to publish to FME Flow. Generally, if your connection is not already on FME Flow, you'll want to include it. You will learn more about managing connections on FME Flow in **the next module**. 

The last section prompts you to select the services to register the workspace.

Transformation Services

The most common services are Data Download, Data Streaming, and Job Submitter, which are transformation services. Transformation services control how the workspace runs and how the written data is output. 

The Job Submitter service allows FME Flow to run a workspace as-is, which is the closest thing to running a workspace in FME Workbench. All inputs and outputs are defined in the workspace, so data is written out as specified in the writers and not streamed or delivered in any other manner. The Data Download and Data Streaming services will be covered in **the last two lessons of this course**. 

You may register a workspace with multiple services, and you will select which service to use when running the workspace on FME Flow. While it is possible to publish a workspace registered to no services, it will not be able to run on FME Flow without registering it with a service, so you should always select at least one service. You may also change the services a workspace is registered with on FME Flow after publishing. 

Publishing

After you've configured all settings using the publishing wizard, clicking Publish transfers the workspace to FME Flow. The translation log in FME Workbench will provide information on whether publishing is successful or not. The Publish Summary denotes information about the recently published workspace and includes a direct link to open the workspace on FME Flow. 

Republishing

The option to republish a workspace to FME Flow appears after publishing has been completed.

You use republish when you've made a few minor modifications to the workspace, such as adding a new transformer or changing Reader settings, and need to publish those changes to FME Flow without modifying any of the original publishing settings. Republish uses the previously used publish settings to publish the workspace to FME Flow without taking you through the publishing wizard. If you need to change any settings in the publishing wizard, such as changing the services a workspace is registered with, republish does not give you those options, and you must go through the entire publishing process. 

Exercise

Sven, a Planning Analyst, is an experienced FME Form user starting to set up his workflows on FME Flow to maximize the FME Platform's capabilities. 

He wants to migrate a workspace that he frequently runs to FME Flow. The workspace reads Parks data from a geodatabase and Neighborhoods data as KML. The Parks are clipped to the neighborhoods, styled for KML, and written as a separate file for each neighborhood using a feature type fanout.  Additionally, an HTML summary report is generated for the Parks data. Sven often shares this workspace with other FME users in his department, so it will be handy to have it on FME Flow where multiple users can access it. 

Keyhole Markup Language (KML): a spatial data format stored in XML, as originally defined by Keyhole, Inc, and now used by Google Earth and specified as an international standard of the Open Geospatial Consortium.

For this exercise, follow along with Sven's steps as he publishes the workspace from FME Workbench to FME Flow. 

1) Open Starting Workspace

Sven opens the starting workspace (C:\FMEData\Workspaces\DeployWorkflowsWithFMEFlow\publish-a-workspace-to-fme-flow.fmw) in FME Workbench (2025.0.1 or later). 

2) Run Workspace and Inspect Output

Sven runs the workspace by clicking the Run button to confirm it functions as expected.

He checks the translation log to confirm that the data is written and the translation is successful.

The output KML Parks data is written to C://FMEData/Output, and he opens the KML files in Google Earth. 

3) Publish to FME Flow

Sven returns to FME Workbench. In the top toolbar, he confirms his FME Flow is connected and clicks Publish to begin the publishing process. 

Sven creates and selects the Training repository to house the workspace and leaves the workspace name as is. Sven could alter the workspace name if he needed to. 

Sven clicks next and registers the workspace with the Job Submitter service for now. Later, Sven will update the workspace and publish it again with the Data Download and Data Streaming services. 

Sven clicks Publish to complete the publishing process. 

3) Check Translation Log

Sven checks the translation log for any errors when publishing the workspace to FME Flow. Sven sees that publishing was successful and reviews the Publish Summary information. 

Sven has successfully published his first workspace to FME Flow. Sven and his colleagues can now access and run this workspace on FME Flow, which he will do in the next lesson.
